# リモートデプロイ設定ガイド

GitHub Actions を使い、`main` ブランチへの Push で自動的に Cloudflare (Workers & Pages) へデプロイするための設定手順です。

## 1. Cloudflare で API トークンを発行する
GitHub があなたの代わりに Cloudflare を操作するための「合鍵」を作ります。

1.  Cloudflare ダッシュボードの右上のアイコンから **「マイプロファイル」** を選択。
2.  左メニューの **「API トークン」** → **「トークンを作成」** をクリック。
3.  **「Cloudflare Workersを編集する」** のテンプレートを使用。
4.  「アカウントリソース」と「ゾーンリソース」を適切に選択（「すべてのアカウント」でも可）して **「続行」**。
5.  表示されたトークンをコピーして控えておく。
    > [!CAUTION]
    > ※一度しか表示されないので注意！

## 2. アカウント ID を確認する
あなたの Cloudflare アカウントの識別番号を確認します。

- Cloudflare ダッシュボードの「ホーム」画面で、右側のサイドバーにある **「アカウント ID」** をコピーします。

## 3. GitHub Secrets に登録する
取得した「合鍵」と「ID」を GitHub リポジトリに安全に保管します。

1.  GitHub リポジトリの **Settings** タブを開く。
2.  左メニューの **Secrets and variables** → **Actions** を選択。
3.  **New repository secret** をクリックして、以下の2つを登録します。

| Secret Name | 説明 |
| :--- | :--- |
| **CLOUDFLARE_API_TOKEN** | 手順1で発行したトークン |
| **CLOUDFLARE_ACCOUNT_ID** | 手順2で確認したアカウント ID |
| **VITE_API_URL** | バックエンド（Workers）のURL（例: `https://my-app-backend.yourname.workers.dev`） |

## 4. 自動デプロイの実行
あとはコードを書いて Push するだけです。

```bash
git add .
git commit -m "feat: 初めての自動デプロイ"
git push origin main
```
GitHub の **Actions** タブで、デプロイが進行する様子をリアルタイムで監視できます。

---

> [!WARNING]
> **注意事項**
> 初回デプロイ時は、Cloudflare Pages のプロジェクトがまだ作成されていないため、エラーになる場合があります。その場合は一度だけローカルから `npx wrangler pages deploy packages/frontend/dist --project-name=プロジェクト名` を実行して、プロジェクトを初期作成してください。
