## 文書の正本

実装と文書が異なる場合は、事故防止のためデプロイせず差異を解消します。APIの入力制約は `packages/shared/src/schemas/index.ts`、DB構造は `packages/backend/migrations/`、Cloudflare bindingは `packages/backend/wrangler.toml` が実装上の正本です。

## 要確認の機密ファイル

実名・学籍番号を含んでいた `doc/dump.sql` は、appからSP D1への初回同期と件数一致を確認後、2026-07-18に作業ツリーから削除しました。過去にリモートへpush済みの場合はGit履歴には残るため、リポジトリ管理者が履歴削除の要否を判断してください。

旧画面キャプチャ7枚は個人情報混入の可能性があり、現行文書から参照されていなかったため、2026-07-18に作業ツリーと全commitから除去しました。

## 更新ルール

- 実在する氏名、学籍番号、メール、認証情報を文書やテストデータへ載せない。
- コマンド例のDB名は必ず `yojitsu-sp-db` とし、appの `yojitsu-db` にSP用migrationを適用しない。
- 完了していないデプロイや運用試験を「完了」と記録しない。
- 日付、実施者、結果を実運用テストチェックリストへ残す。
